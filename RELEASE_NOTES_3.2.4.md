# Nexora 3.2.4 — Update and MLS delivery recovery

Nexora 3.2.4 is a patch release focused on the Windows update path, Server operator console and MLS device recovery.

## Fixed

- Client automatic updates initialize before the renderer can query them and continue checking on schedule.
- The “Проверить обновления” action receives a terminal state even when Electron Updater does not emit one.
- Network failures and releases without signed updater metadata are reported with stable, understandable states.
- Server console commands preserve error codes instead of exposing Electron IPC wrapper text.
- Arguments copied from help, including `plus grant <netrox> [1]`, are normalized safely for every Pulse lookup.
- A verified device without local group state can request MLS Welcome from active members and retry automatically. This shared path covers text, encrypted media and voice messages.

## Added

- On the first 3.2.4 launch, Nexora shows a short release summary with “Подробнее”, “Закрыть” and “Не показывать снова”. “Подробнее” opens the matching GitHub release page.
- `--test-mode` opens a live Windows PowerShell console tailing `nexora-client.log` while the Client remains open.
- The installer creates a “Nexora Client (Test Mode)” Start Menu shortcut.
- Client and Server NSIS installers use Nexora icons, a branded 164×314 sidebar and Russian installer language.

## Operator notes

- Automatic update checks are intentionally disabled in unpackaged development mode. Validate them with an installed packaged Client.
- The production update channel is the GitHub Releases feed for `Onmaynec/Nexora`; installable updates still require the signed NSIS assets and updater metadata generated by the release workflow.
- For console commands, angle and square brackets shown by `help` are placeholders. Both `plus grant netrox 1` and copied input such as `plus grant <netrox> [1]` are accepted.
- MLS Welcome recovery needs at least one verified active group device online. If no active member is available, the Client keeps the existing explicit pending-state error instead of weakening E2EE.

## Security and compatibility

- Update integrity and Authenticode release gates remain enabled; unsigned updater assets are not silently trusted.
- The MLS Welcome request contains no key material. The server only notifies verified devices already active in the group; an active client still creates the RFC 9420 Welcome.
- Local Server schema 8, API v3 and Trust/MLS API v4 are unchanged; no database migration is required.
