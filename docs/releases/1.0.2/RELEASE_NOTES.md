# Nexora 1.0.2 — Connection Reliability

## Главное исправление

Исправлен сценарий со скриншотов: после сверки отпечатка и нажатия «Доверять и подключиться» Client 1.0.1 мог завершить навигацию ошибкой `ERR_CERT_AUTHORITY_INVALID (-202)`.

Причина находилась в границе Node/Electron: предварительная HTTPS-проверка получала SHA-256 листового сертификата, а Chromium мог передать обработчику другое представление поля fingerprint. Из-за этого уже подтверждённый сертификат не всегда совпадал при загрузке интерфейса.

В 1.0.2:

- SHA-256 вычисляется непосредственно из PEM через `X509Certificate`;
- закреплённый сертификат принимается через `session.setCertificateVerifyProc`;
- обработчик `certificate-error` сохранён как безопасный fallback;
- доверие действует только для активного подтверждённого адреса и отпечатка;
- повторное подтверждение сверяет Server ID и тот же SHA-256, исключая замену между двумя проверками;
- смена сервера переустанавливает verifier и отзывает активное доверие.

## Адрес и диагностика

- сокращение `https://26.` больше не превращается URL-парсером в `https://0.0.0.26`;
- требуется полный IPv4 локальной сети/Radmin VPN;
- если порт не указан, Client использует 3443;
- проверяются SAN сертификата, health-ответ, Server ID и fingerprint;
- адрес сохраняется в форме после неудачной навигации;
- ошибки порта, таймаута, маршрута, Radmin VPN и сертификата объясняются отдельно;
- в окне подключения добавлен короткий checklist.

## Правила сертификата

Nexora Client `.exe` не требует установки локального CA в Windows: пользователь сверяет и подтверждает SHA-256 конкретного сервера. Edge/Chrome не используют это хранилище Client, поэтому для браузерного режима корневой `.crt` по-прежнему необходимо установить в доверенные корневые центры.

## Обновление с 1.0.1

1. Установите Client 1.0.2.
2. При сохранённой ошибочной записи нажмите `×` рядом с сервером.
3. Скопируйте полный адрес из Nexora Server, например `https://26.4.1.76:3443`.
4. Сверьте SHA-256 с окном Server и подтвердите подключение.
5. Если Server был запущен до появления Radmin-адаптера, перезапустите Server.

Server 1.0.2 совместим с Client 1.0.x и сохраняет существующую SQLite-базу без миграции схемы.

## Проверено

- `npm run check`;
- 33/33 автоматических теста;
- самоподписанный HTTPS health probe;
- SAN и PEM SHA-256;
- точный регрессионный сценарий `-202`;
- нагрузка 20 клиентов / 120 сообщений;
- SQLite crash recovery и резервные копии.

Подписанные Windows `.exe` требуют сертификата владельца и финальной проверки на Windows 10/11.
