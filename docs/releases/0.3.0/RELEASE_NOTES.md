# Nexora 0.3.0 — объединённый Windows Release Candidate

Дата: 16 июля 2026 года.

Этот релиз объединяет ранее запланированные 0.2.0 «Надёжность данных» и 0.3.0 «Пользователи и интерфейс чатов».

## Надёжность данных

- JSON заменён на SQLite schema 3.
- Legacy `nexora.json` мигрирует автоматически и сохраняется как датированная `.bak`-копия.
- Созданы отдельные таблицы пользователей, сессий, комнат, сообщений, файлов и связанных сущностей.
- Включены WAL и `synchronous=FULL`; логическая запись сообщения/вложения транзакционна.
- `integrity_check` выполняется до загрузки базы.
- Автоматические backup каждые 24 часа; по умолчанию остаются последние 7.
- Ручные «Создать резервную копию» и «Восстановить из копии» находятся в Nexora Server.
- Перед restore автоматически создаётся `pre-restore` copy.
- Добавлены orphan cleanup, file retention, общая quota и статистика занятого места.
- Комнаты экспортируются из серверной панели.

## Аккаунты

- Аватар JPG/PNG/WEBP/GIF до 5 МБ.
- Пользовательский статус до 80 символов.
- Смена пароля с завершением остальных сессий.
- Просмотр и завершение отдельных активных сессий.
- Завершение всех сессий.
- Удаление контакта без блокировки и без удаления истории DM.
- Сброс пароля администратором сервера с немедленным отзывом сессий.

## Сообщения и интерфейс

- Глобальный поиск и поиск в текущем чате.
- Черновики per-user/per-chat.
- Пересылка сообщений и вложений.
- Режим выбора и копирования нескольких сообщений.
- Переход к исходному сообщению ответа через API `around`.
- Разделитель непрочитанных и кнопка перехода вниз.
- Mute отдельного чата и четыре варианта notification sound.
- Offline queue для текста и пересылок со stable client ID.
- Ручный retry/discard failed message; сервер защищён от дублей уникальным индексом.
- Исправлена ошибка голосовых `Rv is not a constructor`: browser `File` больше не перекрывается Lucide-иконкой.
- Hover dock остаётся под логотипом и не перекрывает composer.
- Chat, login и служебные страницы используют согласованный Violet Grid/Aether background.
- Экран входа получил более устойчивую типографику на 1920×1080.

## Автоматическая проверка

```powershell
npm ci
npm run release:check
npm audit --omit=dev
```

Ожидаемый результат:

- syntax — OK;
- Electron Builder config Client/Server — OK;
- Vite production build — OK;
- tests — `23 passed, 0 failed`;
- production audit — `0 vulnerabilities`.

В crash-test отдельный Node-процесс принудительно завершается `SIGKILL` во время транзакции с 20 000 сообщениями. После перезапуска проверяются `PRAGMA integrity_check`, пользователь и история.

## Сборка на Windows

```powershell
npm run dist:windows
```

Ожидаемые установщики:

- `release\client\Nexora-Client-Setup-0.3.0.exe`;
- `release\server\Nexora-Server-Setup-0.3.0.exe`.

## Обязательный Windows smoke-test

1. Собрать установщики на Windows 10/11 и проверить отсутствие builder errors.
2. Установить оба приложения на чистом профиле Windows 10.
3. Повторить базовый запуск на Windows 11.
4. Обновить существующий сервер 0.1.1 с непустым `nexora.json`; проверить созданные `.sqlite` и `.bak`.
5. Запустить Nexora Server, увидеть `Целостность подтверждена`, адрес Radmin и автоматическую копию.
6. Подключить второй ПК через Radmin VPN, установить `.crt`, открыть браузер и Nexora Client.
7. Проверить регистрацию, вход, аватар, статус, смену пароля и список двух сессий.
8. Проверить текст, reply/jump, edit, reaction, pin, search, forward и multi-copy.
9. Остановить сервер, отправить текст offline, запустить сервер и убедиться, что сообщение отправилось один раз.
10. Вызвать failed retry, проверить повтор и удаление из очереди.
11. Записать и отправить голосовое 3–5 секунд; ошибка `constructor` не должна появляться.
12. Проверить mute чата, Windows notification и каждый notification sound.
13. Создать backup, отправить новое сообщение, восстановить backup и проверить откат сообщения и вложений.
14. Установить малую quota и убедиться, что лишняя загрузка отклоняется без orphan-файла.
15. Экспортировать комнату и открыть JSON.
16. Проверить layout 900×620, 1280×820 и 1920×1080; dock и search не перекрывают composer.
17. Удалить приложения штатным uninstall и проверить ярлыки.

## Известные ограничения

- `.exe` не подписаны коммерческим code-signing сертификатом.
- Финальная Windows-сборка и smoke-test должны выполняться на Windows; Linux CI проверяет исходники, web, SQLite и builder schema.
- E2EE отсутствует; владелец локального сервера имеет доступ к данным.
- Offline queue хранит текст и пересылки; бинарные файлы и голос требуют активного соединения.
- Сервер предназначен для закрытой Radmin VPN-сети, а не для прямой публикации в интернет.
