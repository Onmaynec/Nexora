MESSAGE DELIVERY PATH

От composer до verified device.

Каждый переход показывает реальную границу ответственности, а не маркетинговый рейтинг.

  1. 01ComposeClient validates input
  2. 02EncryptMLS application ciphertext
  3. 03AuthorizeSession, device, membership, epoch
  4. 04PersistCiphertext + delivery state
  5. 05DeliverVerified member devices only
  6. 06DecryptLocally on recipient device

TRUST
DEVICE TRUST LIFECYCLE

Идентичность устройства становится частью доступа.

  1. 01
    Identity key

    Client создаёт non-extractable Ed25519 identity key.

  2. 02
    Proof of possession

    Registration подтверждает владение private key.

  3. 03
    Verification

    Дополнительное устройство получает подписанное подтверждение.

  4. 04
    Scoped delivery

    Ciphertext доставляется конкретным verified devices.

  5. 05
    Revocation

    Отзыв немедленно отключает secure socket и очищает local state.

СИЛЬНЫЕ СТОРОНЫ

Контроль и полнота контура

  • Local-first authorityСообщения, комнаты и файлы не требуют централизованного Cloud.
  • Server-side enforcementРоли, bans, media restrictions и realtime access проверяются сервером.
  • Cross-platform UIWindows, Browser/PWA и Android используют общий React-контур.
  • Open sourceMIT разрешает использование, изменение и распространение.
!
ТЕКУЩИЕ ГРАНИЦЫ

Prerelease и metadata visibility

  • Independent auditЛиния 3.2.x не заявляется как независимо аудированная E2EE-система.
  • Traffic metadataСервер видит membership, timing, IP context, ciphertext size и delivery events.
  • Windows signingStable promotion зависит от Authenticode и packaged runtime gates.
  • CallsVoice/video calls и screen sharing не входят в текущий релиз.
