# Руководство тестера Nexora 3.0.0

## Подключение

1. Установите Radmin VPN и подключитесь к сети владельца Nexora.
2. Установите подписанный Nexora Client 3.0.0, PWA или Android-клиент.
3. Скопируйте из окна Server **полный** адрес вида `https://26.x.x.x:3443`.
4. Сверьте SHA-256 и Server ID с владельцем.
5. Нажмите «Доверять и подключиться» и зарегистрируйтесь.

Для `.exe` установка `.crt` не требуется. Для Edge/Chrome/PWA и Android при локальном CA корневой `.crt` обязателен. Никогда не обходите предупреждение о сертификате.

## Если Client не подключается

- `ERR_CERT_AUTHORITY_INVALID (-202)` — удалите сохранённую запись сервера, снова вставьте полный адрес и подтвердите актуальный SHA-256; убедитесь, что установлен Client 3.0.0.
- «Введите полный IPv4-адрес» — `https://26.` недостаточно, нужны четыре октета и порт.
- «Сертификат не содержит адрес» — владелец должен запустить Radmin VPN и перезапустить Server.
- «Нет маршрута/сервер недоступен» — оба ПК должны быть онлайн в одной VPN; проверьте private firewall TCP 3443.
- «Несовместимая версия» — Server API v3 принимает Client major 2–3; обновите более старый Client.

## Приёмочный тест 3.0.0

### Профили и контакты

- Нажмите аватар в сообщении, заголовке, списке чатов, контактах, поиске и room members — везде должна открываться одна карточка профиля.
- Повторите тест для профиля без контакта/общих комнат: вместо пустого экрана должна открыться карточка с доступными действиями.
- Проверьте display name, @username, bio/status, общие комнаты и действия «Написать/Добавить/Блокировать».
- Загрузите аватар, измените пароль, просмотрите и завершите отдельную сессию.
- Удалите контакт без блокировки; история должна сохраниться.

### Чаты

- При отсутствии непрочитанных рядом с «Сообщения» не должно быть `0`.
- Отправьте текст, ответ, реакцию, отредактируйте, удалите, закрепите и сохраните сообщение.
- Наведите мышь на сообщение, откройте реакции и переведите курсор на emoji: picker не должен исчезнуть.
- Откройте details drawer и повторите действия у левого/правого края: меню не должно выходить за чат или прятаться под drawer.
- Проверьте Saved Messages, пересылку, multi-select/copy, drafts, archive/pin/mute и фильтры.
- Проверьте silent/scheduled send, poll, mention, ветку ответа и историю редактирования.
- Отключите Server, прочитайте кэш, поставьте текст в очередь, перезапустите Client, запустите Server и проверьте delta sync без дублей.

### Поиск и непрочитанные

- Выполните глобальный поиск и поиск внутри чата.
- Нажмите результат ответа/поиска — лента должна перейти к исходному сообщению и подсветить его.
- Проверьте разделитель «Новые сообщения» и кнопку «К последнему».

### Комнаты и модерация

- Создайте публичную и приватную комнаты; войдите напрямую, по заявке и по коду.
- Назначьте модератора, передайте владение, удалите/забаньте пользователя.
- Проверьте read-only, slow mode, запрет файлов/голосовых и room audit.
- Выпустите два приглашения, отзовите одно, задайте expiry/usage limit.
- Проверьте custom role/category, жалобу, апелляцию, временное ограничение и pre-approval.

### Файлы и голосовые

- Перетащите несколько файлов; отмените одну загрузку и повторите неудачную.
- Прервите загрузку файла больше 2 МБ и убедитесь, что chunk upload продолжается; поддельное расширение изображения должно быть отклонено/понижено до binary.
- Откройте изображение, PDF/text preview, media tab и общий архив файлов.
- Запишите голосовое: pause → resume → preview → send.
- Проверьте waveform, 1×/1.5×/2×, listened state и продолжение воспроизведения после смены чата.

### Nexora Plus / Pulse

- В обычном `disabled` режиме UI должен объяснять, что Billing не настроен, а бесплатные чаты продолжают работать.
- В специальном `sandbox`-сервере активируйте Plus: баланс должен стать 400, premium frame/accent — доступными.
- Создайте room goal и внесите импульсы; повтор запроса с тем же idempotency key не должен списать баланс второй раз.
- Production checkout проверяется только на отдельном тестовом Billing/provider окружении, без реальных карт в обычном QA.

### Responsive/UI

- Проверьте 1920×1080, 1366×768 и узкое окно.
- Раскройте каждый пункт dock: он не должен выходить за левую панель.
- Проверьте длинные имена, filename, bio и 99+ unread — текст должен обрезаться, а layout не ломаться.

### PWA и Android

- Установите PWA, перезапустите её без сети и убедитесь, что доступен кэш последней истории, но API-ответы не выдаются из Service Worker.
- Откройте Android deep link `nexora://connect?url=<HTTPS URL>`; HTTP URL должен быть отклонён.
- При недоверенном/изменённом сертификате Android обязан закрыть загрузку, а не показывать страницу.
- Ссылка на другой origin должна открыться во внешнем браузере.

### TOTP, боты и эксплуатация

- Включите TOTP, сохраните recovery codes, войдите с кодом и один раз — recovery code; повторное использование должно быть отклонено.
- Создайте bot token только с `messages:write`, отправьте сообщение и убедитесь, что чтение/чужая комната недоступны.
- Проверьте HMAC webhook на публичном HTTPS endpoint; localhost/private IP должен быть отклонён.
- Включите emergency read-only: чтение работает, изменяющие операции блокируются; затем выключите режим.

## Голосовое не записывается

Проверьте HTTPS, доверие сертификату, разрешение микрофона Windows и отсутствие другого приложения, занявшего устройство. В комнате голосовые может отключить модератор. Максимум — 5 минут и 25 МБ.

## Отчёт об ошибке

Укажите версии Client/Server, платформу/OS, точные шаги, ожидаемый/фактический результат, время, состояние сети и приложите скриншот/журнал. Не отправляйте пароль, session cookie, TOTP/recovery code, bot/Pulse API key, invite code, приватный CA key или пароль backup.
