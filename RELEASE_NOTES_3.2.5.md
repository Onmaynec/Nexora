# Nexora 3.2.5 — плавность, медиа и надёжность

Nexora 3.2.5 — исправляющий релиз для проблем, обнаруженных при установленном Windows-клиенте 3.2.4.

## ⚡ Быстрые сообщения без рывков

- история чата больше не очищается и не загружается заново после каждого глобального обновления bootstrap;
- отправка использует локальную зашифрованную очередь и сразу освобождает composer;
- realtime-событие обновляет сообщение и превью чата локально, без повторной загрузки всего bootstrap;
- автоматическая прокрутка срабатывает только для собственного сообщения или когда пользователь уже находится у нижней границы;
- строки сообщений мемоизированы и не перерисовываются при каждом вводе символа.

## 🔐 MLS Welcome и синхронизация

- гонка одновременного создания MLS-группы и временное отсутствие подходящего KeyPackage переходят в защищённый запрос Welcome и ограниченное ожидание вместо немедленной ошибки;
- синхронизация повторяется в фоне, поэтому composer не блокирует пользователя длительным сетевым ожиданием;
- для уже проверенной локальной epoch добавлен короткий безопасный fast path без отключения серверной валидации или plaintext fallback;
- запрос от нового устройства по-прежнему заставляет активного участника выполнить полную синхронизацию и создать Welcome.

## 🖼️ Изображения и голосовые

- изображения автоматически расшифровываются локально и отображаются прямо внутри сообщения;
- голосовые получили компактный плеер с play/pause, waveform, перемоткой, длительностью и скоростью воспроизведения;
- обычные файлы сохраняют явное локальное открытие и скачивание;
- сервер по-прежнему получает только ciphertext и opaque media.

## 🛠️ Nexora Server

- исправлено несоответствие `localUserId`/`userId`, из-за которого `plus grant` и `impulses grant` передавали `undefined` в SQLite;
- добавлен интеграционный regression-тест с настоящим `node:sqlite`;
- disabled-кнопки, поля, полосы прокрутки и command output приведены к визуальному языку Nexora.

## ✨ Интерфейс

- системное Windows-окно обновления заменено на доступное модальное окно внутри клиента;
- интерактивная сеть отображается только в истории чата, реагирует на курсор и не попадает в шапку, боковую панель, composer или другие разделы;
- окно обновления поддерживает «Подробнее», «Закрыть», Escape и «Не показывать снова».

## 📦 Windows-сборка

- `npm run release:windows` создаёт локальные тестовые установщики без требования сертификата;
- `npm run release:windows:signed` сохраняет обязательную проверку Authenticode;
- официальный workflow публикации по-прежнему блокирует `.exe`, `.blockmap` и `latest.yml`, если секреты подписи отсутствуют.

## Совместимость

- Local Server schema: `8`, без миграции;
- Application API: v3, без breaking changes;
- Trust/MLS API: v4, обратно совместим;
- обновление поддерживается с 3.2.0–3.2.4.
