# Nexora 0.1.1 — Windows Release Candidate

Дата кандидата: 16 июля 2026 года.

## Что изменено

- Hover-меню перенесено из нижнего центра в верхнюю часть левой панели, сразу под логотип Nexora. Оно больше не перекрывает строку ввода, подсказки и вложения.
- Исправлена цепочка голосовых сообщений: Windows Client выдаёт разрешение на аудио только доверенным локальным HTTPS-адресам, запись ожидает ответ сервера, а превью сохраняется при ошибке для повторной отправки.
- Для `.webm`, `.ogg` и `.m4a` выбирается корректное расширение; новое сообщение сразу добавляется в открытый чат и затем синхронизируется через bootstrap/Socket.IO.
- В чате включён более плотный интерактивный фон Aether/Violet Grid с радиусом реакции 200 px. На служебных экранах остаётся спокойный режим, чтобы не мешать чтению.
- Служебные страницы ограничены по полезной ширине, нижние отступы после переноса меню убраны, а в настройках отображается номер Windows RC.
- Nexora Server сохраняет `nexora-server.log` между запусками и умеет открыть его из раздела «Журнал».
- API `/api/health` и `/api/bootstrap` возвращают версию `0.1.1`.

## Автоматическая проверка

Перед сборкой установщиков выполните:

```powershell
npm ci
npm run release:check
npm audit --omit=dev
```

Ожидаемый результат для этого RC:

- web build Vite — успешно;
- конфигурации Electron Builder Client/Server — успешно;
- интеграционные тесты — `11 passed, 0 failed`;
- production audit — `0 vulnerabilities`.

## Сборка Windows

На Windows 10 или 11:

```powershell
npm run dist:windows
```

Ожидаемые файлы:

- `release\client\Nexora-Client-Setup-0.1.1.exe`;
- `release\server\Nexora-Server-Setup-0.1.1.exe`.

## Обязательный smoke-test установщиков

1. Установить оба приложения на чистый профиль Windows 10 или 11.
2. Запустить Nexora Server и проверить адрес `https://26.x.x.x:3443`, QR-код и экспорт CA.
3. Установить CA у тестера, запустить Nexora Client и подключиться по Radmin VPN.
4. Зарегистрировать два аккаунта, принять заявку в контакты и открыть личный чат.
5. Записать голосовое 3–5 секунд, прослушать превью, отправить и воспроизвести у второго пользователя.
6. Повторить голосовое в общей комнате и проверить появление в разделе «Файлы».
7. Проверить, что hover-меню раскрывается под логотипом и нигде не перекрывает composer.
8. Проверить фон частиц в чате, открытие details drawer и размеры окна 900×620, 1280×820 и 1920×1080.
9. Открыть постоянный журнал сервера из панели и убедиться, что он содержит строку запуска.
10. Удалить приложения через Windows и убедиться, что ярлыки удалены корректно.

## Известные ограничения RC

- Установщики не подписаны коммерческим code-signing сертификатом, поэтому SmartScreen может показать предупреждение.
- Автоматизированная проверка покрывает сервер, web build и конфигурации упаковки; финальный smoke-test `.exe` выполняется на Windows.
- Хранилище остаётся локальным JSON и рассчитано на тестовую VPN-сеть.
- Сквозного шифрования нет; HTTPS защищает транспорт, но владелец серверного компьютера контролирует данные.
